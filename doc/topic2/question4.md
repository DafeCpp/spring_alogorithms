# Сортировка с помощью кучи

## 1. Идея heap sort

`Heap sort` строится на двоичной куче и использует тот факт, что в `max-heap`
корень всегда содержит максимум.

Сортировка по неубыванию делается так:

1. строим `max-heap`;
2. меняем корень с последним элементом;
3. уменьшаем рабочую часть массива;
4. восстанавливаем кучу;
5. повторяем.

## 2. Почему это работает

На каждом шаге максимум текущего диапазона ставится в самый правый ещё не
зафиксированный элемент.

После этого:

- этот элемент уже стоит на правильном месте;
- остаётся отсортировать оставшуюся часть.

## 3. Связь с кучей

Очень полезно понять, что `heap sort` — это не отдельная магия, а прямое
использование очереди с приоритетом.

Если бы мы:

- вставили все элементы в кучу;
- много раз извлекали максимум,

мы получили бы отсортированную последовательность.

`Heap sort` просто делает это in-place в массиве.

## 4. Почему построение кучи стоит `O(n)`, а не `O(n log n)`

Это один из красивых фактов.

Хотя отдельное `sift down` стоит до `O(log n)`, массовое построение снизу вверх
даёт суммарно линейное время.

## 5. Полная сложность

- построение кучи: `O(n)`;
- `n` извлечений по `O(log n)`;
- итог: `O(n log n)`.

## 7. Реализация на C++

```cpp
void SiftDown(std::vector<int>& a, int root, int heap_size) {
  while (true) {
    int left = 2 * root + 1;
    int right = 2 * root + 2;
    int largest = root;

    if (left < heap_size && a[left] > a[largest]) {
      largest = left;
    }
    if (right < heap_size && a[right] > a[largest]) {
      largest = right;
    }
    if (largest == root) {
      break;
    }
    std::swap(a[root], a[largest]);
    root = largest;
  }
}

void HeapSort(std::vector<int>& a) {
  for (int i = static_cast<int>(a.size()) / 2 - 1; i >= 0; --i) {
    SiftDown(a, i, static_cast<int>(a.size()));
  }

  for (int heap_size = static_cast<int>(a.size()); heap_size > 1; --heap_size) {
    std::swap(a[0], a[heap_size - 1]);
    SiftDown(a, 0, heap_size - 1);
  }
}
```

## 8. Сводная таблица свойств

| Алгоритм | Лучший случай | Худший случай | Доп. память | Узнать длину | Взять максимум | Добавить элемент | Устойчивость |
|---|---|---|---|---|---|---|---|
| Heap sort | `O(n log n)` | `O(n log n)` | `O(1)` | `O(1)` | `O(1)` | `O(n)` | нет |

## 9. Плюсы

- сортировка `in-place`;
- хороший худший случай `O(n log n)`;
- не требует дополнительного массива размера `n`.

## 10. Минусы

- неустойчива;
- на практике часто медленнее quick sort;
- локальность памяти и константы хуже, чем у некоторых конкурентов.

## 11. Когда о ней нужно помнить

Когда:

- важен худший случай `O(n log n)`;
- нужна сортировка без дополнительной памяти;
- хочется использовать кучу как основу решения.

## 12. Что важно запомнить

`Heap sort` — это сортировка через многократное извлечение максимума из кучи,
реализованная прямо внутри массива.
