# MSD, LSD. Сортировка строк

## 1. Два направления поразрядной сортировки

Есть два базовых подхода:

- `LSD` — Least Significant Digit, обработка справа налево;
- `MSD` — Most Significant Digit, обработка слева направо.

Обе идеи относятся к поразрядной сортировке, но ведут себя по-разному.

## 2. LSD

### Как работает

Сначала сортируем по самому младшему разряду, затем по следующему, и так далее.

### Когда особенно удобен

- все ключи одинаковой длины;
- нужна простая реализация;
- можно использовать устойчивую сортировку на каждом проходе.

### Плюсы

- простая идея;
- очень естественно сочетается с counting sort;
- хорошо работает для фиксированных целочисленных ключей.

### Минусы

- плохо подходит для сильно различающихся длин строк без дополнительных трюков.

### Реализация на C++

Ниже — `LSD` для строк одинаковой длины над байтовым алфавитом.

```cpp
void LSDSortStrings(std::vector<std::string>& a, int width) {
  const int kAlphabet = 256;
  std::vector<std::string> output(a.size());

  for (int pos = width - 1; pos >= 0; --pos) {
    std::vector<int> count(kAlphabet + 1, 0);

    for (const auto& s : a) {
      ++count[static_cast<unsigned char>(s[pos]) + 1];
    }
    for (int i = 1; i <= kAlphabet; ++i) {
      count[i] += count[i - 1];
    }
    for (const auto& s : a) {
      output[count[static_cast<unsigned char>(s[pos])]++] = s;
    }
    a = output;
  }
}
```

## 3. MSD

### Как работает

Сначала сортируем по старшему разряду, затем рекурсивно сортируем получившиеся
группы по следующему разряду.

### Когда особенно удобен

- строки могут иметь разную длину;
- старшие символы сильно делят данные на естественные группы;
- нужен более “лексикографический” стиль обработки.

### Плюсы

- очень естественен для строк;
- быстро отсеивает большие группы по первым символам.

### Минусы

- реализация сложнее;
- сложнее аккуратно работать с маленькими подзадачами и разной длиной строк.

### Реализация на C++

Ниже — упрощённая рекурсивная идея `MSD` для строк.

```cpp
int CharAt(const std::string& s, int pos) {
  return pos < static_cast<int>(s.size())
             ? static_cast<unsigned char>(s[pos])
             : -1;
}

void MSDSort(std::vector<std::string>& a, int left, int right, int pos) {
  if (left >= right) {
    return;
  }

  const int kAlphabet = 256;
  std::vector<int> count(kAlphabet + 2, 0);
  std::vector<std::string> aux(right - left + 1);

  for (int i = left; i <= right; ++i) {
    ++count[CharAt(a[i], pos) + 2];
  }
  for (int i = 1; i < static_cast<int>(count.size()); ++i) {
    count[i] += count[i - 1];
  }

  std::vector<int> start = count;
  for (int i = left; i <= right; ++i) {
    aux[count[CharAt(a[i], pos) + 1]++] = a[i];
  }
  for (int i = left; i <= right; ++i) {
    a[i] = aux[i - left];
  }

  for (int r = 0; r < kAlphabet; ++r) {
    int bucket_left = left + start[r];
    int bucket_right = left + start[r + 1] - 1;
    if (bucket_left <= bucket_right) {
      MSDSort(a, bucket_left, bucket_right, pos + 1);
    }
  }
}
```

## 4. Почему тема особенно важна для строк

Для строк первый символ часто даёт очень много информации.

Пример:

```text
cat
car
dog
door
```

Уже по первой букве видно разделение на группы `c...` и `d...`.

Именно поэтому `MSD`-подход для строк часто выглядит очень естественным.

## 5. Сравнение

| Подход | Идея | Сильная сторона |
|---|---|---|
| LSD | справа налево | фиксированная длина ключа |
| MSD | слева направо | строки и лексикографический порядок |

## 6. Сводные таблицы свойств

| Алгоритм | Лучший случай | Худший случай | Доп. память | Узнать длину | Взять максимум | Добавить элемент | Устойчивость |
|---|---|---|---|---|---|---|---|
| LSD string sort | `O(w(n + k))` | `O(w(n + k))` | `O(n + k)` | `O(1)` | `O(1)` | `O(n)` | да, если проходы устойчивы |
| MSD string sort | близко к `O(w n)` на хороших данных | до `O(w n)`–`O(w n + k * levels)` в зависимости от реализации | `O(n + k)` | `O(1)` | `O(1)` | `O(n)` | зависит от реализации |

## 7. Что важно запомнить

LSD и MSD — это не конкурирующие “версии одного и того же”, а два разных стиля
поразрядной сортировки, каждый из которых лучше подходит к своему типу данных.
