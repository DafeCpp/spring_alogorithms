# Асимптотические обозначения и работа с ними

## 1. Зачем вообще нужна асимптотика

Когда мы сравниваем алгоритмы, нас обычно интересует не время на одном
конкретном запуске, а то, как поведение алгоритма растёт при увеличении размера
входа.

Например:

- для массива из 10 элементов почти любой алгоритм работает быстро;
- для массива из `10^6` элементов разница между `O(n)` и `O(n^2)` становится
  колоссальной.

Асимптотика — это язык, на котором мы описываем именно **скорость роста**
затрат.

## 2. Что такое размер входа

Перед тем как говорить о сложности, нужно понять, что мы считаем за `n`.

Чаще всего:

- для массива это длина массива;
- для строки — длина строки;
- для графа — число вершин и/или рёбер;
- для матрицы — размеры по строкам и столбцам.

Очень важно правильно выбрать параметр размера. Иначе оценка может оказаться
бессмысленной.

## 3. Big O — верхняя оценка

Запись:

```text
T(n) = O(f(n))
```

означает, что начиная с достаточно больших `n` функция `T(n)` растёт не быстрее
константы, умноженной на `f(n)`.

Интуитивно это “не хуже чем”.

### Примеры

- один проход по массиву — `O(n)`;
- двойной вложенный цикл по `n` — `O(n^2)`;
- бинарный поиск — `O(log n)`;
- merge sort — `O(n log n)`.

## 4. Omega — нижняя оценка

Запись:

```text
T(n) = Ω(f(n))
```

означает, что функция не может расти медленнее этого порядка.

Интуитивно это “не лучше чем”.

### Пример

Чтобы хотя бы прочитать все элементы массива длины `n`, уже нужно `Ω(n)`
операций. Значит никакой алгоритм, которому обязательно надо посмотреть каждый
элемент, не сможет работать быстрее линейного времени.

## 5. Theta — точная оценка по порядку роста

Запись:

```text
T(n) = Θ(f(n))
```

означает, что сверху и снизу функция ведёт себя одинаково по порядку роста.

Это значит:

- и `O(f(n))`;
- и `Ω(f(n))`.

Интуитивно: “растёт именно как”.

## 6. Частые классы сложности

| Класс | Что означает | Примеры |
|---|---|---|
| `O(1)` | не зависит от `n` | доступ по индексу |
| `O(log n)` | очень медленный рост | бинарный поиск |
| `O(n)` | линейный рост | линейный поиск |
| `O(n log n)` | почти линейно, но тяжелее | merge sort, heap sort |
| `O(n^2)` | квадратичный рост | insertion sort, selection sort |
| `O(2^n)` | экспоненциальный рост | наивные переборы |

## 7. Как упрощают формулы

В асимптотике обычно:

- отбрасывают константы;
- отбрасывают младшие члены;
- смотрят на главный рост.

### Примеры

```text
3n + 7          -> O(n)
5n^2 + 2n + 1  -> O(n^2)
n log n + n     -> O(n log n)
1000            -> O(1)
```

## 8. Почему константы всё же не бесполезны

В теории `O(n)` лучше, чем `O(n log n)`.
Но на маленьких входах алгоритм с худшей асимптотикой может быть быстрее из-за:

- маленькой константы;
- лучшей локальности памяти;
- меньшей сложности кода.

Поэтому на практике асимптотика — это не всё, но это очень важная первая
ориентировка.

## 9. Время и память

Говоря об асимптотике, нужно различать:

- **временную сложность** — сколько операций выполняется;
- **пространственную сложность** — сколько дополнительной памяти используется.

Например:

- merge sort: `O(n log n)` по времени и `O(n)` по памяти;
- heap sort: `O(n log n)` по времени и `O(1)` по памяти.

## 10. Лучший, средний и худший случаи

Один и тот же алгоритм может иметь разные оценки:

- лучший случай;
- средний случай;
- худший случай.

Пример:

- insertion sort на почти отсортированном массиве работает быстро;
- на обратном порядке — медленно.

Поэтому важно понимать, **о какой именно оценке идёт речь**.

## 11. Как правильно использовать асимптотику

Хороший порядок мышления такой:

1. понять, что такое размер входа;
2. оценить число базовых операций;
3. упростить формулу;
4. сравнить порядок роста с альтернативами.

## 12. Что важно запомнить

Асимптотика — это язык роста алгоритмов. Она не говорит всё о производительности
на практике, но позволяет быстро отличать:

- масштабируемые решения;
- решения, которые разваливаются на больших входах.
