# ДП на матрицах. Кол-во путей в графе из u в v длины ровно k

## 1. Почему эта задача — прямое продолжение Фибоначчи

После матричного представления Фибоначчи появляется очень важная следующая
мысль:

> матрица может описывать не только рекуррентную последовательность, но и
> переходы по графу.

Если в Фибоначчи матрица кодировала переход между двумя последовательными
состояниями, то здесь она кодирует:

- из какой вершины;
- в какую вершину;
- можно перейти за один шаг.

И тогда возведение матрицы в степень начинает отвечать на вопрос:

> что происходит за много шагов подряд?

## 2. Постановка задачи

Дан ориентированный граф с вершинами `0..n-1`.
Нужно узнать, сколько существует путей:

- из вершины `u`;
- в вершину `v`;
- ровно длины `k`.

Под длиной пути здесь обычно понимается число рёбер.

## 3. Матрица смежности

Пусть `A` — матрица смежности графа.

Тогда:

```text
A[i][j] = 1, если ребро i -> j существует
A[i][j] = 0, если ребра нет
```

Если граф — мультиграф, элемент матрицы может хранить число рёбер между
вершинами. Взвешенные варианты тоже допускают обобщения, но в базовой версии
мы работаем с обычной матрицей смежности.

## 4. Что означает один шаг

Матрица `A` описывает все переходы длины 1.

То есть:

```text
A[u][v]
```

говорит, сколько существует способов попасть из `u` в `v` ровно за один шаг.

В обычном невзвешенном графе это просто `0` или `1`.

## 5. Что означает квадрат матрицы

Посмотрим на:

```text
A^2[u][v]
```

По формуле умножения матриц:

```text
A^2[u][v] = sum A[u][t] * A[t][v]
```

Что означает каждое слагаемое?

- `A[u][t]` — есть ли путь из `u` в `t` за один шаг;
- `A[t][v]` — есть ли путь из `t` в `v` за один шаг.

Их произведение отвечает на вопрос:

> можно ли пройти из `u` в `v` через промежуточную вершину `t` за два шага?

Сумма по всем `t` и даёт количество путей длины 2.

## 6. Почему степени матрицы считают пути

Индуктивно получается:

```text
A^k[u][v] = количество путей из u в v длины ровно k
```

Это один из самых красивых фактов в алгоритмах и линейной алгебре.

Причина в том, что матричное умножение делает именно то, что нужно динамике по
числу шагов:

- перебирает все промежуточные состояния;
- аккуратно суммирует число способов перейти через них.

## 7. Связь с обычной динамикой по длине пути

Можно было бы ввести:

```text
dp[len][v] = количество путей длины len,
             оканчивающихся в вершине v
```

Тогда переход:

```text
dp[len + 1][to] += dp[len][from]
```

по всем рёбрам `from -> to`.

Матричное умножение делает ровно это же, только:

- сразу для всех вершин;
- в компактной векторно-матричной форме.

То есть это не другая идея, а та же динамика, записанная более мощным языком.

## 8. Почему здесь полезно быстрое возведение в степень

Если считать `A^k` обычным перемножением `k` раз, будет долго.

Но матрицу можно возводить в степень бинарным методом:

- `A^1`, `A^2`, `A^4`, `A^8`, ...
- собираем нужную степень по битам `k`.

Это даёт:

```text
O(log k)
```

матричных умножений.

## 9. Итоговая сложность

Если граф имеет `n` вершин и мы перемножаем обычные квадратные матрицы
наивно, то одно умножение стоит:

```text
O(n^3)
```

Тогда итог:

```text
O(n^3 log k)
```

Для больших `k` это очень полезно, потому что наивная динамика по всем длинам
до `k` стоила бы:

```text
O(k n^3)
```

или `O(km)` в других реализациях.

## 10. Матричная форма через вектор состояния

Можно мыслить ещё так:

- есть вектор `dp_t`, где `dp_t[v]` — число способов оказаться в вершине `v`
  после `t` шагов;
- тогда:

```text
dp_{t+1} = dp_t * A
```

или `A * dp_t` — в зависимости от соглашения о строках/столбцах.

После `k` шагов:

```text
dp_k = dp_0 * A^k
```

Это делает связь с Фибоначчи ещё более явной:

- там тоже был вектор состояния;
- там тоже был фиксированный переход;
- там тоже нужна была степень матрицы.

## 11. Небольшой пример

Пусть граф:

```text
0 -> 1
0 -> 2
1 -> 2
2 -> 1
```

Тогда матрица смежности:

```text
0 1 1
0 0 1
0 1 0
```

Тогда:

- `A[0][1] = 1` — один путь длины 1;
- `A^2[0][1]` покажет число путей длины 2 из 0 в 1;
- например, путь `0 -> 2 -> 1` даёт вклад в эту клетку.

## 12. Реализация на C++

```cpp
using Matrix = std::vector<std::vector<long long>>;

Matrix Multiply(const Matrix& a, const Matrix& b) {
  int n = static_cast<int>(a.size());
  Matrix c(n, std::vector<long long>(n, 0));
  for (int i = 0; i < n; ++i) {
    for (int k = 0; k < n; ++k) {
      if (a[i][k] == 0) {
        continue;
      }
      for (int j = 0; j < n; ++j) {
        c[i][j] += a[i][k] * b[k][j];
      }
    }
  }
  return c;
}

Matrix Power(Matrix base, long long exp) {
  int n = static_cast<int>(base.size());
  Matrix result(n, std::vector<long long>(n, 0));
  for (int i = 0; i < n; ++i) {
    result[i][i] = 1;
  }

  while (exp > 0) {
    if (exp & 1LL) {
      result = Multiply(result, base);
    }
    base = Multiply(base, base);
    exp >>= 1LL;
  }

  return result;
}

long long CountPathsExactlyK(const Matrix& adjacency, int u, int v, long long k) {
  Matrix p = Power(adjacency, k);
  return p[u][v];
}
```

## 13. Почему это всё ещё динамика

Очень важно проговорить:

мы не “ушли из динамики в линейную алгебру”, а просто переписали динамику в
матричный язык.

Состояние здесь:

- распределение количества способов по вершинам;
- или вся таблица переходов между вершинами.

Переход:

- один шаг по графу.

Степень матрицы:

- многократное применение одного и того же перехода.

## 14. Какие задачи эта идея помогает узнавать

Если в задаче встречаются формулировки:

- “ровно за `k` шагов”;
- “ровно за `k` переходов”;
- “сколько способов пройти длину `k`”;
- “переход между состояниями фиксирован и линейный”,

очень стоит подумать о матрице переходов.

## 15. Типичные ошибки

- путать путь длины `k` с путём из `k` вершин;
- перепутать соглашение о строках и столбцах;
- забыть, что для длины 0 нужна единичная матрица;
- считать, что `A^k` отвечает за пути длины “не более `k`”, хотя это только
  ровно `k`;
- ошибиться в типе данных при больших количествах путей.

## 16. Что важно запомнить

Матрица смежности в степени `k` — это очень мощная и красивая форма динамики по
длине пути.

Она учит видеть:

1. состояние как распределение по вершинам;
2. переход как шаг по графу;
3. степень матрицы как многократное применение одного и того же перехода.
