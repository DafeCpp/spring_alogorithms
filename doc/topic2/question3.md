# Сортировка слиянием

## 1. Почему merge sort считается фундаментальной

`Merge sort` — один из самых важных алгоритмов сортировки в курсе, потому что он
одновременно показывает:

- идею “разделяй и властвуй”;
- гарантированную асимптотику `O(n log n)`;
- мощь аккуратного линейного слияния.

## 2. Главная идея

Алгоритм работает так:

1. делим массив пополам;
2. рекурсивно сортируем левую половину;
3. рекурсивно сортируем правую половину;
4. сливаем две отсортированные половины.

## 3. Почему разбиение не решает задачу само по себе

Если просто разбить массив, проблема не исчезает. Она распадается на две такие
же проблемы меньшего размера. Это и есть суть “divide and conquer”.

## 4. Слияние — ключевая операция

Пусть у нас уже есть:

```text
left  = [2, 5, 8]
right = [1, 3, 7]
```

Можно поставить два указателя в начало и каждый раз брать меньший элемент:

```text
1 -> 2 -> 3 -> 5 -> 7 -> 8
```

Вся операция выполняется за линейное время относительно суммы длин половин.

## 6. Почему сложность именно `O(n log n)`

На каждом уровне рекурсии все слияния суммарно обрабатывают `n` элементов.

Число уровней рекурсии:

```text
log n
```

Значит итог:

```text
O(n log n)
```

## 7. Память

Классическая версия `merge sort` использует дополнительный массив для слияния.
Поэтому:

- время: `O(n log n)`;
- дополнительная память: `O(n)`.

## 8. Устойчивость

`Merge sort` легко сделать устойчивой.

Если текущие элементы слева и справа равны, можно брать сначала элемент из
левой половины, и тогда относительный порядок равных элементов сохранится.

## 9. Реализация на C++

```cpp
void Merge(std::vector<int>& a, int left, int mid, int right) {
  std::vector<int> temp;
  temp.reserve(right - left + 1);

  int i = left;
  int j = mid + 1;

  while (i <= mid && j <= right) {
    if (a[i] <= a[j]) {
      temp.push_back(a[i++]);
    } else {
      temp.push_back(a[j++]);
    }
  }

  while (i <= mid) {
    temp.push_back(a[i++]);
  }
  while (j <= right) {
    temp.push_back(a[j++]);
  }

  for (int k = 0; k < static_cast<int>(temp.size()); ++k) {
    a[left + k] = temp[k];
  }
}

void MergeSort(std::vector<int>& a, int left, int right) {
  if (left >= right) {
    return;
  }
  int mid = left + (right - left) / 2;
  MergeSort(a, left, mid);
  MergeSort(a, mid + 1, right);
  Merge(a, left, mid, right);
}
```

## 10. Сводная таблица свойств

| Алгоритм | Лучший случай | Худший случай | Доп. память | Узнать длину | Взять максимум | Добавить элемент | Устойчивость |
|---|---|---|---|---|---|---|---|
| Merge sort | `O(n log n)` | `O(n log n)` | `O(n)` | `O(1)` | `O(1)` | `O(n)` | да |

## 11. Плюсы

- гарантированная хорошая асимптотика;
- устойчивость;
- очень хорошо подходит для списков;
- естественно используется во внешней сортировке.

## 12. Минусы

- нужна дополнительная память;
- на маленьких входах бывают быстрее более простые алгоритмы;
- рекурсивная реализация не всегда самая дешёвая по константам.

## 13. Где используется

- сортировка больших массивов и файлов;
- внешний merge sort на диске;
- подсчёт числа инверсий;
- стандартные библиотечные реализации и гибриды.

## 14. Что важно запомнить

Сила `merge sort` не в хитрой формуле, а в сочетании двух идей:

- задача делится пополам;
- две отсортированные последовательности можно слить линейно.
