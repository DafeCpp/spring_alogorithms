# Поиск k-ой порядковой статистики методом QuickSelect

## 1. Что такое порядковая статистика

`k`-я порядковая статистика — это элемент, который стоял бы на позиции `k`
после сортировки массива.

Примеры:

- `k = 1` — минимум;
- `k = n` — максимум;
- середина — медиана.

## 2. Почему сортировать весь массив бывает лишним

Если нам нужен только один элемент, полная сортировка тратит силы на порядок
между всеми остальными элементами, хотя нас этот порядок может вообще не
интересовать.

Отсюда естественный вопрос:

> можно ли найти нужный элемент быстрее, чем за `O(n log n)`?

## 3. Идея QuickSelect

Алгоритм очень похож на quick sort:

1. выбираем `pivot`;
2. делаем `partition`;
3. смотрим, на какую позицию встал `pivot`;
4. идём только в ту часть, где точно лежит нужный ответ.

## 4. Почему это быстрее

Quick sort рекурсирует в обе стороны.
QuickSelect рекурсирует только в одну.

То есть после каждого разбиения мы выбрасываем значительную часть массива из
дальнейшего рассмотрения.

## 5. Интуиция

Если после partition опорный элемент оказался:

- слишком левым — нужная статистика правее;
- слишком правым — нужная статистика левее;
- ровно на позиции `k` — ответ найден.

## 6. Сложность

- средняя: `O(n)`;
- худшая: `O(n^2)`.

Средняя линейность — именно то, почему QuickSelect так важен.

## 7. Где используется

- поиск медианы;
- статистики по потоку или выборке;
- алгоритмы top-k;
- задачи, где нужен пороговый элемент, а не весь порядок.

## 8. Что важно запомнить

QuickSelect показывает очень важную идею:

для ответа на вопрос о порядке не всегда нужно полностью сортировать массив.
