# Сортировка подсчётом. Карманная сортировка

## Почему эти алгоритмы особенные

Они часто работают быстрее сравнительных сортировок, потому что используют не
только сравнения, а дополнительную информацию о значениях элементов.

## Сортировка подсчётом

### Когда подходит

Если элементы — целые числа из небольшого диапазона `0..k`.

### Идея

1. считаем, сколько раз встретилось каждое значение;
2. по массиву частот восстанавливаем отсортированный ответ.

Пример:

```text
[2, 1, 2, 0, 1]
```

Частоты:

```text
0 -> 1
1 -> 2
2 -> 2
```

Ответ:

```text
[0, 1, 1, 2, 2]
```

### Визуализация

![Counting sort](./img/Counting%20sort.gif)

### Сложность

- время: `O(n + k)`;
- память: `O(k)`.

### Реализация на C++

```cpp
std::vector<int> CountingSort(const std::vector<int>& a, int max_value) {
  std::vector<int> count(max_value + 1, 0);

  for (int x : a) {
    ++count[x];
  }

  std::vector<int> result;
  result.reserve(a.size());

  for (int value = 0; value <= max_value; ++value) {
    while (count[value] > 0) {
      result.push_back(value);
      --count[value];
    }
  }

  return result;
}
```

### Сводная таблица свойств

| Алгоритм | Лучший случай | Худший случай | Доп. память | Узнать длину | Взять максимум | Добавить элемент | Устойчивость |
|---|---|---|---|---|---|---|---|
| Counting sort | `O(n + k)` | `O(n + k)` | `O(k)` | `O(1)` | `O(1)` | `O(n)` | да, в устойчивой версии |

## Карманная сортировка (bucket sort)

### Идея

1. делим область значений на корзины;
2. раскладываем элементы по корзинам;
3. сортируем элементы внутри каждой корзины;
4. собираем корзины обратно.

### Когда хороша

Если данные распределены достаточно равномерно, средний размер корзины будет
маленьким, и сортировка работает очень быстро.

### Реализация на C++

```cpp
void BucketSort(std::vector<double>& a) {
  int n = static_cast<int>(a.size());
  std::vector<std::vector<double>> buckets(n);

  for (double x : a) {
    int bucket = std::min(n - 1, static_cast<int>(x * n));
    buckets[bucket].push_back(x);
  }

  for (auto& bucket : buckets) {
    std::sort(bucket.begin(), bucket.end());
  }

  int pos = 0;
  for (const auto& bucket : buckets) {
    for (double x : bucket) {
      a[pos++] = x;
    }
  }
}
```

### Сводная таблица свойств

| Алгоритм | Лучший случай | Худший случай | Доп. память | Узнать длину | Взять максимум | Добавить элемент | Устойчивость |
|---|---|---|---|---|---|---|---|
| Bucket sort | `O(n + k)` при хорошем распределении | `O(n^2)` в плохом случае | `O(n + k)` | `O(1)` | `O(1)` | `O(n)` | зависит от сортировки внутри корзин |

## Сравнение

| Алгоритм | Когда хорош |
|---|---|
| Counting sort | маленький диапазон значений |
| Bucket sort | удачное распределение по корзинам |

## Ограничения

Эти алгоритмы не универсальны: они выигрывают только на специальных типах
данных.
