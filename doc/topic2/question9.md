# Поразрядная сортировка

## 1. Главная идея

Поразрядная сортировка (`radix sort`) сортирует элементы не целиком, а по
разрядам:

- цифрам числа;
- символам строки;
- позициям в ключе фиксированной длины.

Это делает её принципиально отличной от сравнительных сортировок.

## 2. Почему это может быть быстрее `O(n log n)`

Нижняя оценка `Ω(n log n)` относится только к сортировкам сравнением.
Поразрядная сортировка использует дополнительную информацию о строении ключа, а
значит может обойти это ограничение.

## 3. Как работает один проход

На каждом шаге сортируются элементы по одному конкретному разряду:

- по единицам;
- по десяткам;
- по сотням;
- и так далее.

Очень важно, чтобы внутренний проход был **устойчивым**.

## 4. Почему устойчивость критична

Если сортировка по текущему разряду неустойчива, порядок, построенный на
предыдущих разрядах, может разрушиться.

Именно поэтому radix sort почти всегда строится поверх устойчивой сортировки,
например counting sort.

## 5. Для каких данных особенно удобна

- целые числа фиксированной длины;
- строки одинаковой длины;
- коды, идентификаторы, номера;
- данные с небольшим алфавитом или маленьким диапазоном цифр.

## 6. Сложность

Если:

- `d` — число разрядов;
- `k` — размер алфавита или число возможных значений разряда,

то сложность:

```text
O(d * (n + k))
```

## 7. Реализация на C++

Ниже приведён классический `LSD radix sort` для неотрицательных целых чисел в
десятичной системе.

```cpp
void CountingSortByDigit(std::vector<int>& a, int exp) {
  std::vector<int> count(10, 0);
  std::vector<int> output(a.size());

  for (int x : a) {
    int digit = (x / exp) % 10;
    ++count[digit];
  }

  for (int i = 1; i < 10; ++i) {
    count[i] += count[i - 1];
  }

  for (int i = static_cast<int>(a.size()) - 1; i >= 0; --i) {
    int digit = (a[i] / exp) % 10;
    output[count[digit] - 1] = a[i];
    --count[digit];
  }

  a = output;
}

void RadixSort(std::vector<int>& a) {
  int max_value = *std::max_element(a.begin(), a.end());
  for (int exp = 1; max_value / exp > 0; exp *= 10) {
    CountingSortByDigit(a, exp);
  }
}
```

## 8. Сводная таблица свойств

| Алгоритм | Лучший случай | Худший случай | Доп. память | Узнать длину | Взять максимум | Добавить элемент | Устойчивость |
|---|---|---|---|---|---|---|---|
| Radix sort | `O(d(n + k))` | `O(d(n + k))` | `O(n + k)` | `O(1)` | `O(1)` | `O(n)` | да, если внутренние проходы устойчивы |

## 9. Когда метод особенно силён

Если `d` невелико и фиксировано, а `k` небольшой, radix sort может работать
очень быстро и реально побеждать многие сравнительные методы.

## 10. Ограничения

- алгоритм не универсален для произвольных объектов;
- нужно уметь естественно извлекать разряды;
- эффективность зависит от длины ключа и размера алфавита.

## 11. Что важно запомнить

Поразрядная сортировка выигрывает не потому, что “волшебнее” quick sort, а
потому что использует специальную структуру ключа и сортирует его по частям.
