# Квадратичные сортировки: сортировка вставками, выбором

## Почему их называют квадратичными

Такие сортировки в худшем случае делают порядка `n^2` сравнений или действий.
Они медленнее современных `O(n log n)` алгоритмов на больших входах, но очень
важны для понимания базовых идей.

## Как читать сравнительные таблицы ниже

Для колонок:

- “узнать длину”;
- “взять максимум”;
- “добавить элемент”

будем считать, что после сортировки результат хранится в обычном
`std::vector<int>`.

Тогда:

- длина берётся через `size()` за `O(1)`;
- максимум в отсортированном по неубыванию массиве — последний элемент, значит
  `O(1)`;
- добавление нового элемента **с сохранением отсортированности** требует
  найти позицию и сдвинуть хвост, поэтому обычно это `O(n)`.

## Сортировка вставками

### Интуиция

Представь, что мы раскладываем карты в руке. Левая часть уже отсортирована, а
очередную карту нужно вставить в правильное место.

### Алгоритм

1. считаем, что первый элемент уже отсортирован;
2. берём следующий элемент;
3. сдвигаем вправо все большие элементы слева;
4. вставляем элемент на освободившееся место.

```cpp
void InsertionSort(std::vector<int>& a) {
  for (int i = 1; i < static_cast<int>(a.size()); ++i) {
    int x = a[i];
    int j = i - 1;
    while (j >= 0 && a[j] > x) {
      a[j + 1] = a[j];
      --j;
    }
    a[j + 1] = x;
  }
}
```

### Плюсы

- проста;
- устойчива;
- очень хороша на почти отсортированных данных;
- часто используется внутри гибридных алгоритмов.

### Минусы

- на больших случайных данных работает медленно;
- много сдвигов элементов.

## Сортировка выбором

### Интуиция

На каждом шаге ищем минимум неотсортированной части и ставим его на следующее
готовое место.

### Алгоритм

1. ищем минимальный элемент на отрезке `i..n-1`;
2. меняем его местами с `a[i]`;
3. повторяем для следующей позиции.

```cpp
void SelectionSort(std::vector<int>& a) {
  for (int i = 0; i < static_cast<int>(a.size()); ++i) {
    int min_pos = i;
    for (int j = i + 1; j < static_cast<int>(a.size()); ++j) {
      if (a[j] < a[min_pos]) {
        min_pos = j;
      }
    }
    std::swap(a[i], a[min_pos]);
  }
}
```

### Плюсы

- очень проста;
- делает мало обменов.

### Минусы

- неустойчива в классическом виде;
- всегда работает примерно как `O(n^2)`.

## Сравнение

| Алгоритм | Лучший случай | Худший случай | Доп. память | Узнать длину | Взять максимум | Добавить элемент | Устойчивость |
|---|---|---|---|---|---|---|---|
| Insertion sort | `O(n)` | `O(n^2)` | `O(1)` | `O(1)` | `O(1)` | `O(n)` | да |
| Selection sort | `O(n^2)` | `O(n^2)` | `O(1)` | `O(1)` | `O(1)` | `O(n)` | нет |

## Когда они всё ещё полезны

- маленькие массивы;
- обучение и понимание базовых идей сортировки;
- как база для гибридных алгоритмов.
