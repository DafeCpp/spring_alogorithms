# Задача о наибольшей возрастающей подпоследовательности

## 1. Постановка и терминология

Пусть дан массив:

```text
a[0], a[1], ..., a[n-1]
```

Нужно найти длину наибольшей возрастающей подпоследовательности (`LIS`,
`Longest Increasing Subsequence`).

Подпоследовательность — это элементы массива, взятые в исходном порядке, но не
обязательно подряд.

Если выбраны индексы:

```text
i1 < i2 < ... < ik
```

то соответствующая подпоследовательность возрастающая, если:

```text
a[i1] < a[i2] < ... < a[ik]
```

## 2. Почему задача важна

`LIS` — одна из центральных задач динамического программирования и алгоритмов в
целом. На ней очень хорошо видно:

- как строится базовая квадратичная динамика;
- как появляется смысл “состояние оканчивается в `i`”;
- как потом эту динамику можно ускорить;
- как структуры данных и бинарный поиск помогают улучшить решение.

Это важная учебная веха: здесь мы уже уходим от совсем простых `dp[i] =
dp[i-1] + ...` и начинаем думать о более содержательной зависимости.

## 3. Почему жадный подход здесь опасен

Можно придумать наивные идеи:

- всегда брать минимально возможный следующий элемент;
- всегда брать ближайший больший;
- всегда продолжать текущую цепочку, если это возможно.

Но такие локальные решения не гарантируют глобального оптимума.

Причина в том, что удачность выбора элемента зависит не только от текущего
момента, но и от того, насколько он оставляет пространство для продолжения в
будущем.

Именно поэтому задача естественно тянет в сторону динамики.

## 4. Базовая динамика `O(n^2)`

### 4.1. Выбор состояния

Самый важный шаг:

```text
dp[i] = длина LIS, которая оканчивается в a[i]
```

Почему именно так, а не просто “лучшая длина на префиксе до `i`”?

Потому что для продолжения последовательности критично знать **последний
элемент**. Если мы знаем, что цепочка оканчивается в `a[i]`, мы можем проверить,
подходит ли следующий элемент для продолжения.

### 4.2. База

Каждый элемент сам по себе образует возрастающую подпоследовательность длины 1:

```text
dp[i] = 1
```

### 4.3. Переход

Если `j < i` и `a[j] < a[i]`, то к последовательности, оканчивающейся в `j`,
можно приписать `a[i]`.

Тогда:

```text
dp[i] = 1 + max(dp[j]) по всем j < i, где a[j] < a[i]
```

Если таких `j` нет, остаётся база:

```text
dp[i] = 1
```

### 4.4. Почему это корректно

Любая возрастающая подпоследовательность, оканчивающаяся в `a[i]`, должна иметь
предыдущий элемент `a[j]`, где:

- `j < i`;
- `a[j] < a[i]`.

Значит достаточно перебрать все допустимые предыдущие концы и выбрать лучший.

Это очень характерный тип рассуждения в `DP`:

> оптимальный ответ для состояния строится через выбор последнего шага.

## 5. Пошаговый пример для `O(n^2)`

Пусть:

```text
a = [3, 1, 5, 2, 6, 4, 9]
```

Тогда:

- `dp[0] = 1` (`[3]`)
- `dp[1] = 1` (`[1]`)
- `dp[2] = 2` (`[3, 5]` или `[1, 5]`)
- `dp[3] = 2` (`[1, 2]`)
- `dp[4] = 3` (`[3, 5, 6]` или `[1, 5, 6]`)
- `dp[5] = 3` (`[1, 2, 4]`)
- `dp[6] = 4` (`[1, 2, 4, 9]` или `[1, 5, 6, 9]`)

Ответ:

```text
max(dp[i]) = 4
```

Очень важно заметить:

ответ не обязан оканчиваться в последнем элементе массива.

## 6. Почему итоговый ответ — это максимум по всем `dp[i]`

`dp[i]` описывает последовательности, оканчивающиеся в конкретной позиции.
Но исходная задача не фиксирует конец.

Поэтому:

```text
LIS = max(dp[i]) по всем i
```

Это один из самых типичных паттернов:

- состояние отвечает за “ответ при фиксированном последнем объекте”;
- итоговый ответ берётся как максимум по всем таким концам.

## 7. Восстановление самой подпоследовательности

Если нужно не только число, но и сама `LIS`, удобно хранить:

```text
prev[i] = индекс предыдущего элемента
          в лучшей подпоследовательности, оканчивающейся в i
```

Когда мы улучшаем `dp[i]` через `j`, делаем:

```text
prev[i] = j
```

Потом:

1. находим индекс `best`, где `dp[best]` максимально;
2. идём назад по `prev`;
3. разворачиваем полученный список.

## 8. Реализация `O(n^2)` на C++

```cpp
std::vector<int> GetLisQuadratic(const std::vector<int>& a) {
  const int n = static_cast<int>(a.size());
  if (n == 0) {
    return {};
  }

  std::vector<int> dp(n, 1);
  std::vector<int> prev(n, -1);

  int best_index = 0;

  for (int i = 0; i < n; ++i) {
    for (int j = 0; j < i; ++j) {
      if (a[j] < a[i] && dp[j] + 1 > dp[i]) {
        dp[i] = dp[j] + 1;
        prev[i] = j;
      }
    }
    if (dp[i] > dp[best_index]) {
      best_index = i;
    }
  }

  std::vector<int> lis;
  for (int v = best_index; v != -1; v = prev[v]) {
    lis.push_back(a[v]);
  }
  std::reverse(lis.begin(), lis.end());
  return lis;
}
```

## 9. Сложность квадратичного решения

- внешний цикл по `i`: `O(n)`;
- внутренний цикл по `j < i`: в сумме `O(n^2)`;
- память: `O(n)`.

Это уже вполне рабочее решение для умеренных ограничений.

## 10. Переход к более быстрому решению

Квадратичное решение хорошо для понимания, но иногда медленно.

Теперь появляется более тонкая идея.

### 10.1. Что именно тормозит

Для каждого `i` мы ищем:

```text
максимальный dp[j], где j < i и a[j] < a[i]
```

То есть слишком дорого перебираем все предыдущие позиции.

Нужно научиться хранить информацию компактнее.

## 11. Идея “лучших хвостов”

Пусть:

```text
d[len] = минимальный возможный хвост
         возрастающей подпоследовательности длины len
```

Смысл этой конструкции:

- если мы умеем закончить подпоследовательность длины `len` маленьким хвостом,
  то это лучше, чем большим хвостом;
- маленький хвост легче продолжить в будущем.

Это одна из самых красивых интуиций во всей теме `LIS`.

## 12. Почему минимальный хвост — это хорошо

Сравним две подпоследовательности одинаковой длины:

- одна заканчивается числом `10`;
- другая заканчивается числом `4`.

Вторая объективно “перспективнее”, потому что больше будущих чисел смогут её
продолжить.

Поэтому для каждой длины нам важно помнить **не любую** цепочку, а цепочку с
наилучшим хвостом.

## 13. Свойства массива `d`

Массив `d` монотонен:

```text
d[1] < d[2] < d[3] < ...
```

Почему?

Потому что подпоследовательность большей длины не может заканчиваться числом,
которое не больше хвоста более короткой подпоследовательности.

Именно из-за этой монотонности появляется бинарный поиск.

## 14. Как обрабатывается новый элемент

Для очередного `x = a[i]` мы ищем первую длину `len`, для которой:

```text
d[len] >= x
```

И заменяем `d[len]` на `x`.

Интерпретация:

- мы нашли подпоследовательность длины `len - 1`, которую можно продолжить `x`;
- теперь получили подпоследовательность длины `len` с, возможно, более выгодным
  хвостом.

## 15. Почему это не хранит сам ответ напрямую

Очень важный психологический барьер:

массив `d` **не обязан быть реальной подпоследовательностью массива**.

Он хранит:

- не конкретный ответ;
- а лучшую возможную информацию о хвостах для каждой длины.

То есть это скорее “сводка” о том, какие длины уже достижимы и насколько
хорошими хвостами они заканчиваются.

## 16. Почему здесь работает бинарный поиск

Поскольку `d` отсортирован, можно за `O(log n)` найти место, куда поставить
новый элемент.

Отсюда и получается асимптотика:

```text
O(n log n)
```

## 17. Реализация длины `LIS` за `O(n log n)`

```cpp
int GetLisLength(const std::vector<int>& a) {
  std::vector<int> d;

  for (int x : a) {
    auto it = std::lower_bound(d.begin(), d.end(), x);
    if (it == d.end()) {
      d.push_back(x);
    } else {
      *it = x;
    }
  }

  return static_cast<int>(d.size());
}
```

### Почему `lower_bound`

Для строго возрастающей подпоследовательности нужно найти первое значение,
которое не меньше `x`.

Если бы задача была про неубывающую подпоследовательность, логика границы
менялась бы.

## 18. Восстановление ответа в `O(n log n)`

Это уже более продвинутая часть. Нужно хранить:

- индекс конца лучшей цепочки каждой длины;
- массив предков для восстановления.

То есть быстрое решение может восстанавливать не только длину, но и саму
последовательность, просто реализация становится сложнее.

## 19. Сравнение двух решений

| Подход | Время | Память | Основная идея |
|---|---|---|---|
| `O(n^2)` | `O(n^2)` | `O(n)` | перебираем все предыдущие позиции |
| `O(n log n)` | `O(n log n)` | `O(n)` | храним лучшие хвосты и ищем место бинарным поиском |

## 20. Где `LIS` встречается ещё

Эта задача всплывает в:

- обработке последовательностей;
- задачах на частичный порядок;
- геометрических задачах после сортировки;
- сведении к “удалить минимум элементов, чтобы стало возрастающим”;
- продвинутых задачах с деревом Фенвика и деревом отрезков.

## 21. Типичные ошибки

- путать подпоследовательность с подмассивом;
- брать ответ как `dp[n-1]`, а не `max(dp)`;
- использовать `<=` вместо `<` и случайно решать другую задачу;
- думать, что массив `d` сам является одной реальной подпоследовательностью;
- неверно восстанавливать ответ.

## 22. Что важно запомнить

`LIS` важна по двум причинам:

1. как очень содержательная динамика с состоянием “оканчивается в `i`”;
2. как пример того, как из честной квадратичной динамики рождается более быстрое
   решение через другой взгляд на состояние.
