# Представление очереди в виде двух стеков. Время извлечения элемента

## 1. Почему это интересная идея

На первый взгляд стек и очередь ведут себя противоположно:

- стек — `LIFO`;
- очередь — `FIFO`.

Поэтому особенно красиво, что очередь всё же можно построить **из двух стеков**.

## 2. Конструкция

Используются два стека:

- `in` — сюда кладутся новые элементы;
- `out` — отсюда элементы выдаются наружу.

## 3. Операция добавления

`push(x)` просто кладёт `x` в стек `in`.

Это всегда `O(1)`.

## 4. Операция извлечения

Если стек `out` не пуст, снимаем элемент с его вершины.

Если `out` пуст:

1. переносим все элементы из `in` в `out`;
2. порядок разворачивается;
3. теперь верхушка `out` — это старейший элемент очереди.

## 5. Почему порядок получается правильным

Пусть в `in` лежат:

```text
[1, 2, 3, 4]
```

где `4` — вершина.

После переливания в `out` порядок станет:

```text
[4, 3, 2, 1]
```

А значит на вершине `out` окажется `1`, то есть именно тот элемент, который
должен выйти из очереди первым.

## 6. Почему извлечение не становится дорогим навсегда

Кажется, что операция `pop()` иногда ужасно дорогая, потому что переносит сразу
много элементов.

Но важно смотреть не на одну операцию, а на длинную серию.

Каждый элемент:

- один раз кладётся в `in`;
- максимум один раз переносится в `out`;
- максимум один раз удаляется из `out`.

Значит суммарная работа на элемент ограничена константой.

## 7. Амортизированная оценка

Отсюда получается:

```text
amortized O(1)
```

на операцию добавления и удаления.

Это один из самых красивых базовых примеров амортизационного анализа.

## 8. Зачем это знать

Эта конструкция полезна:

- как учебный пример композиции структур данных;
- как мост к очереди с минимумом через два min-stack;
- как пример того, что “дорогая иногда” не значит “дорогая в среднем”.

## 9. Сводная таблица практических характеристик

| Свойство | Оценка |
|---|---|
| Память | `O(n)` |
| Узнать длину | `O(1)`, если хранить размеры обоих стеков |
| Взять максимум | `O(n)` |
| Добавить элемент `push` | `O(1)` |
| Удалить элемент `pop` | `O(1)` амортизированно |

## 10. Что важно запомнить

Очередь через два стека — это классический пример, где противоположные по
поведению структуры можно скомбинировать и получить нужный интерфейс с хорошей
амортизированной сложностью.
