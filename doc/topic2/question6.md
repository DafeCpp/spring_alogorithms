# Поиск числа инверсий

## Что такое инверсия

Инверсия — это пара индексов `(i, j)`, где:

- `i < j`;
- `a[i] > a[j]`.

Иными словами, элементы стоят не в том порядке, в котором должны стоять после
сортировки.

## Почему это интересная величина

Число инверсий показывает, насколько массив далёк от отсортированного состояния.

- у полностью отсортированного массива инверсий `0`;
- у массива в обратном порядке их максимальное число.

## Наивный способ

Проверить все пары:

```text
for i:
  for j > i:
    if a[i] > a[j]: count++
```

Сложность: `O(n^2)`.

## Идея быстрого решения

Во время `merge sort` можно одновременно сортировать массив и считать инверсии.

Если при слиянии оказалось, что элемент из правой половины меньше текущего
элемента из левой, то он образует инверсии **со всеми оставшимися элементами
левой части**.

## Пример

```text
left  = [2, 5, 8]
right = [1, 3, 7]
```

Когда берём `1` из правой части, он меньше `2`, `5`, `8`, значит добавляем `3`
инверсии сразу.

## Сложность

- время: `O(n log n)`;
- память: `O(n)`.

## Где это применяется

- оценка «почти отсортированности»;
- сравнение перестановок;
- задачи на ранжирование и согласованность порядков.
