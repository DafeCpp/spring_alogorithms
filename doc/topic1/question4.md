# Двусвязный и односвязный список. Операции. Объединение списков

## 1. Что такое связный список

Связный список — это структура данных, состоящая из узлов, где каждый узел
хранит:

- данные;
- ссылки на соседние узлы.

В отличие от массива элементы списка не обязаны лежать подряд в памяти.

## 2. Почему вообще нужны списки

Список полезен, когда:

- размер структуры часто меняется;
- важно быстро вставлять и удалять элементы по краям или рядом с известным
  узлом;
- доступ по индексу не является главным требованием.

## 3. Односвязный список

В односвязном списке у узла есть:

- значение;
- ссылка на следующий узел.

Пример:

```text
[A] -> [B] -> [C] -> null
```

### Плюсы

- простая реализация;
- мало дополнительной памяти по сравнению с двусвязным;
- удобно реализовывать стек.

### Минусы

- нельзя быстро двигаться назад;
- удаление узла сложнее без указателя на предыдущий.

## 4. Двусвязный список

В двусвязном списке у узла есть:

- значение;
- ссылка на следующий;
- ссылка на предыдущий.

Пример:

```text
null <- [A] <-> [B] <-> [C] -> null
```

### Плюсы

- можно двигаться в обе стороны;
- удобно удалять и вставлять узел относительно известной позиции;
- хорошо подходит для дека.

### Минусы

- больше памяти на указатели;
- реализация чуть сложнее.

## 5. Основные операции

### Вставка в начало

Новый узел становится новой головой списка.

### Вставка после узла

Если известен указатель на узел, вставка после него делается за `O(1)`.

### Удаление головы

Просто передвигаем указатель головы.

### Поиск по значению

Требует линейного обхода: `O(n)`.

## 6. Сложности

| Операция | Односвязный | Двусвязный |
|---|---|---|
| Вставка в голову | `O(1)` | `O(1)` |
| Удаление головы | `O(1)` | `O(1)` |
| Поиск | `O(n)` | `O(n)` |
| Переход назад | нет | `O(1)` |

## 7. Объединение списков

Если у нас есть два списка, их можно объединить очень дёшево, если известны:

- хвост первого;
- голова второго.

Тогда достаточно:

- у хвоста первого поменять `next`;
- у головы второго, если список двусвязный, поменять `prev`.

То есть объединение может быть `O(1)` — без копирования всех элементов.

## 8. Сравнение со структурой массива

| Структура | Сильная сторона | Слабая сторона |
|---|---|---|
| Массив | доступ по индексу | дорогие вставки/удаления в середине |
| Список | гибкие вставки/удаления | плохой доступ по индексу |

## 9. Сводная таблица практических характеристик

Здесь важно помнить, что некоторые оценки зависят не только от идеи структуры,
но и от того, какие служебные поля мы решили хранить в реализации.

Например, длину списка можно узнавать:

- за `O(1)`, если мы поддерживаем поле `size`;
- за `O(n)`, если каждый раз считаем элементы обходом.

| Свойство | Односвязный список | Двусвязный список |
|---|---|---|
| Память | `O(n)` узлов + одна ссылка `next` в каждом узле | `O(n)` узлов + две ссылки `prev/next` в каждом узле |
| Узнать длину | `O(1)` с `size`, иначе `O(n)` | `O(1)` с `size`, иначе `O(n)` |
| Взять максимум | `O(n)` | `O(n)` |
| Добавить элемент | `O(1)` в голову; в хвост `O(1)` только если хранится хвост | `O(1)` в голову и в хвост при хранении обоих концов |

## 10. Когда список — правильный выбор

- часто добавляем и удаляем элементы;
- не нужен быстрый доступ по индексу;
- важна гибкость структуры.

## 11. Что важно запомнить

Связный список — это структура, оптимизированная не под индексный доступ, а под
гибкое изменение связей между элементами.
