# Формулировка задачи сортировки. Устойчивость.

## Что такое сортировка

Сортировка — это перестановка элементов массива так, чтобы они шли в нужном
порядке:

- по неубыванию;
- по невозрастанию;
- или по какому-то пользовательскому критерию.

Пример:

```text
[5, 1, 4, 2, 3] -> [1, 2, 3, 4, 5]
```

## Зачем сортировка нужна на практике

После сортировки становится проще:

- искать элементы;
- удалять дубликаты;
- выполнять бинарный поиск;
- строить статистики и медианы;
- сравнивать наборы данных.

Во многих задачах сортировка — это не конечная цель, а подготовительный шаг.

## Что считается корректной сортировкой

Алгоритм сортировки должен:

1. не потерять элементы;
2. не добавить новые элементы;
3. вернуть ту же самую коллекцию, но в правильном порядке.

## Устойчивость

Сортировка называется **устойчивой**, если равные элементы сохраняют свой
взаимный порядок.

Пример:

```text
(2, "A"), (1, "B"), (2, "C")
```

После устойчивой сортировки:

```text
(1, "B"), (2, "A"), (2, "C")
```

Элементы со значением `2` остались в прежнем относительном порядке.

## Почему устойчивость важна

Это особенно полезно, когда элементы имеют несколько полей.

Например:

1. сначала отсортировали людей по имени;
2. потом устойчиво сортируем по возрасту.

Тогда среди людей одного возраста сохранится порядок по имени.

## Какие свойства ещё часто сравнивают

| Свойство | Что означает |
|---|---|
| Устойчивость | равные элементы не переставляются местами без нужды |
| In-place | не требуется много дополнительной памяти |
| Асимптотика | как растёт время работы при увеличении `n` |
| Адаптивность | умеет ли алгоритм использовать почти отсортированный вход |

## Что важно запомнить

Сортировка — это не одна задача и не один алгоритм. Важны сразу несколько
вопросов:

- нужен ли стабильный порядок равных элементов;
- насколько велик вход;
- можно ли тратить дополнительную память;
- важен ли худший случай или достаточно хорошего среднего.
