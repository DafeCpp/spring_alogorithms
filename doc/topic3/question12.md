# Операции вставки и удаления в АВЛ-дереве

## 1. Общая картина

Все операции в AVL-дереве удобно понимать как:

1. обычную операцию `BST`;
2. восстановление AVL-инварианта.

То есть идея поиска по ключу никуда не девается. Новая сложность появляется
только потому, что после изменения дерева нужно убедиться, что локальные
разницы высот снова в допустимых пределах.

---

## 2. Вставка: первый этап

Сначала новая вершина вставляется ровно так же, как в обычном `BST`:

- сравниваем ключ;
- идём влево или вправо;
- доходим до пустого места;
- создаём новую вершину.

До этого момента AVL ещё никак не проявляется.

---

## 3. Почему вставка может сломать баланс

Новая вершина добавляется в листовую позицию.

Это может:

- увеличить высоту некоторого поддерева на 1;
- а значит изменить баланс-фактор у предков на пути к корню.

Важно:

влияют только вершины на пути от места вставки до корня.

Все остальные части дерева вообще не меняются.

---

## 4. Второй этап вставки: подъём вверх

После вставки мы поднимаемся обратно к корню и на каждой вершине:

1. пересчитываем высоту;
2. пересчитываем баланс-фактор;
3. проверяем, не стал ли модуль больше 1.

Если нарушение найдено, делаем соответствующее вращение.

---

## 5. Почему после вставки часто хватает одного исправления

После вставки первая вершина, где нарушился AVL-инвариант, — это самое нижнее
проблемное место.

После корректного вращения структура этого поддерева обычно восстанавливается
так, что дальше вверх дополнительная проблема уже не распространяется.

Это одна из причин, почему AVL-вставка кажется проще удаления.

---

## 6. Четыре случая вставки

Как и в теме про вращения, вставка после нарушения может дать:

- `LL`;
- `RR`;
- `LR`;
- `RL`.

Именно тип локального перекоса определяет, нужно одно вращение или двойное.

Нужно всегда смотреть:

1. куда “тяжела” проблемная вершина;
2. куда “тяжёл” её ребёнок.

---

## 7. Удаление: первый этап

Удаление в AVL начинается как в обычном `BST`:

- находим удаляемую вершину;
- разбираем случаи: 0, 1 или 2 ребёнка;
- делаем базовую BST-перестройку.

И только после этого начинается AVL-специфика.

---

## 8. Почему удаление сложнее вставки

После вставки поддерево может только вырасти.

После удаления поддерево может:

- уменьшить высоту;
- и это уменьшение может “потащить” проблему выше по дереву.

То есть после одного локального исправления высота всё ещё может измениться
так, что предок тоже окажется разбалансирован.

Именно поэтому AVL-удаление потенциально требует серии исправлений по пути
вверх.

---

## 9. Второй этап удаления: подъём и починка

После удаления мы поднимаемся к корню и на каждой вершине:

1. обновляем высоту;
2. смотрим баланс-фактор;
3. при необходимости вращаем.

В отличие от вставки, здесь восстановление может продолжиться на нескольких
уровнях.

Это один из самых важных conceptual differences между AVL-вставкой и
AVL-удалением.

---

## 10. Что обязательно хранит вершина

В практической реализации у вершины обычно есть:

- ключ;
- указатель на левого ребёнка;
- указатель на правого ребёнка;
- высота или баланс-фактор.

Без хранения высоты/баланса пришлось бы каждый раз пересчитывать высоты больших
поддеревьев, и реализация стала бы медленной и неудобной.

---

## 11. Почему все операции всё равно `O(log n)`

Потому что:

- путь от корня до места изменения имеет длину `O(log n)`;
- вращение — локальная `O(1)` операция;
- число точек, которые приходится проверять, пропорционально длине этого пути.

Значит и вставка, и удаление остаются:

```text
O(log n)
```

## 12. Реализация на C++

```cpp
struct Node {
  int key;
  int height = 1;
  Node* left = nullptr;
  Node* right = nullptr;
};

int Height(Node* v) {
  return v == nullptr ? 0 : v->height;
}

void UpdateHeight(Node* v) {
  if (v != nullptr) {
    v->height = 1 + std::max(Height(v->left), Height(v->right));
  }
}

int BalanceFactor(Node* v) {
  return Height(v->right) - Height(v->left);
}

Node* RotateLeft(Node* x) {
  Node* y = x->right;
  Node* t2 = y->left;
  y->left = x;
  x->right = t2;
  UpdateHeight(x);
  UpdateHeight(y);
  return y;
}

Node* RotateRight(Node* y) {
  Node* x = y->left;
  Node* t2 = x->right;
  x->right = y;
  y->left = t2;
  UpdateHeight(y);
  UpdateHeight(x);
  return x;
}

Node* Rebalance(Node* v) {
  UpdateHeight(v);

  if (BalanceFactor(v) == 2) {
    if (BalanceFactor(v->right) < 0) {
      v->right = RotateRight(v->right);
    }
    return RotateLeft(v);
  }
  if (BalanceFactor(v) == -2) {
    if (BalanceFactor(v->left) > 0) {
      v->left = RotateLeft(v->left);
    }
    return RotateRight(v);
  }
  return v;
}

Node* Insert(Node* root, int key) {
  if (root == nullptr) {
    return new Node{key};
  }
  if (key < root->key) {
    root->left = Insert(root->left, key);
  } else if (key > root->key) {
    root->right = Insert(root->right, key);
  }
  return Rebalance(root);
}

Node* FindMin(Node* root) {
  return root->left == nullptr ? root : FindMin(root->left);
}

Node* RemoveMin(Node* root) {
  if (root->left == nullptr) {
    return root->right;
  }
  root->left = RemoveMin(root->left);
  return Rebalance(root);
}

Node* Erase(Node* root, int key) {
  if (root == nullptr) {
    return nullptr;
  }

  if (key < root->key) {
    root->left = Erase(root->left, key);
  } else if (key > root->key) {
    root->right = Erase(root->right, key);
  } else {
    Node* left = root->left;
    Node* right = root->right;
    delete root;

    if (right == nullptr) {
      return left;
    }

    Node* min_node = FindMin(right);
    min_node->right = RemoveMin(right);
    min_node->left = left;
    return Rebalance(min_node);
  }

  return Rebalance(root);
}
```

## 13. Как правильно понимать цену вращений

Иногда у студентов возникает ощущение:

“Если мы крутим дерево, не становится ли это слишком дорогим?”

Нет.

Причина:

- каждое конкретное вращение работает над константным числом вершин;
- дорого не само вращение, а поиск пути до проблемного места;
- но этот путь логарифмический.

То есть вращения не портят асимптотику, а лишь являются локальным инструментом
починки.

---

## 14. Почему AVL считают инженерно нетривиальным

Сама математическая идея компактна:

- вставить/удалить как в BST;
- поправить баланс.

Но реализация требует аккуратности:

- правильно вернуть новый корень поддерева;
- обновить высоты в корректном порядке;
- обработать все 4 случая перекоса;
- не забыть, что после удаления проблема может пойти выше.

Поэтому AVL очень полезен как структура, которая тренирует точное мышление о
рекурсивных изменениях дерева.

---

## 15. Что важно запомнить

AVL-вставка и AVL-удаление — это не “магия вращений”, а обычные BST-операции
плюс фаза восстановления инварианта:

1. найти и изменить нужную позицию как в BST;
2. поднимаясь вверх, обновлять высоты;
3. при необходимости выполнять одно или два вращения.

Самая важная разница между вставкой и удалением:

- после вставки проблема обычно локализуется быстрее;
- после удаления баланс может потребовать исправлений на нескольких уровнях.
