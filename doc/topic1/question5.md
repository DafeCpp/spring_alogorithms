# Стек

## Что такое стек

Стек — это структура данных, которая работает по принципу `LIFO`:

```text
Last In, First Out
```

Последний добавленный элемент извлекается первым.

Простой бытовой образ — стопка тарелок: новую тарелку кладём сверху и забираем
тоже сверху.

## Основные операции

| Операция | Смысл |
|---|---|
| `push(x)` | положить элемент наверх |
| `pop()` | снять верхний элемент |
| `top()` | посмотреть верхний элемент |
| `empty()` | проверить, пуст ли стек |

## Пример работы

```text
push(3) -> [3]
push(7) -> [3, 7]
push(1) -> [3, 7, 1]
pop()   -> [3, 7]
```

## Реализация на массиве

Если стек хранится в массиве, достаточно поддерживать индекс вершины.

Плюсы:

- очень быстро;
- хорошая локальность памяти;
- простая реализация.

Минусы:

- у статического массива фиксированный размер;
- при динамической версии нужны расширения памяти.

## Реализация на списке

Стек удобно хранить и в списке:

- вершина — это голова списка;
- `push` вставляет новый узел в голову;
- `pop` удаляет голову.

## Сложность

Все базовые операции стека выполняются за `O(1)`.

## Сводная таблица практических характеристик

У обычного стека быстрыми являются только операции на вершине. Если мы хотим
уметь быстро брать максимум или минимум по всем элементам, стек нужно
дополнять отдельной идеей вроде `min-stack` или `max-stack`.

| Свойство | Оценка |
|---|---|
| Память | `O(n)` |
| Узнать длину | `O(1)`, если хранится `size`; иначе `O(n)` |
| Взять максимум | `O(n)` |
| Добавить элемент `push` | `O(1)` |

## Где стек используется

- рекурсия и стек вызовов;
- проверка скобочных последовательностей;
- обходы в глубину (`DFS`);
- вычисление выражений;
- undo/redo в приложениях.

## Типичные ошибки

- вызывать `pop` у пустого стека;
- забывать, что стек не умеет быстро доставать элементы из середины;
- путать стек с очередью.

## Что важно запомнить

Стек — одна из самых базовых структур данных, и очень многие алгоритмы можно
увидеть как "обработку элементов с вершины".
