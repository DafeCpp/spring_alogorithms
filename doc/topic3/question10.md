# АВЛ-дерево. Вращения

## 1. Почему вообще понадобились AVL-деревья

Обычное `BST` хранит порядок красиво, но никак не защищает себя от вырождения.

AVL-дерево — это одна из первых строгих идей балансировки:

- дерево остаётся деревом поиска;
- но дополнительно контролирует форму почти в каждой вершине.

Главная философия AVL:

> не допускать даже умеренного локального перекоса.

---

## 2. Что такое AVL-дерево

AVL-дерево — это бинарное дерево поиска, в котором для каждой вершины
выполняется:

```text
|height(left) - height(right)| <= 1
```

То есть высоты левого и правого поддеревьев отличаются не более чем на единицу.

Это очень сильное условие. Оно намного жёстче, чем у red-black дерева.

---

## 3. Почему именно разница высот

Если у вершины одно поддерево систематически заметно выше другого, дерево
начинает вытягиваться.

AVL запрещает накопление такого эффекта.

Каждый локальный узел обязан выглядеть “почти симметрично” с точки зрения
высот, и из большого числа таких локальных ограничений рождается глобальная
логарифмическая высота.

---

## 4. Баланс-фактор

В реализации удобно хранить:

```text
balance(v) = height(right) - height(left)
```

или эквивалентно с противоположным знаком.

Допустимые значения:

- `-1`
- `0`
- `1`

Если модуль больше единицы, вершина разбалансирована.

Баланс-фактор важен потому, что он даёт короткий локальный сигнал:

- нужно ли исправление;
- и в какую сторону перекосилось дерево.

---

## 5. Почему строгий баланс даёт сильный поиск

Чем строже дерево контролирует форму, тем меньше его высота.

AVL — один из самых строгих массово используемых вариантов.

Из-за этого:

- дерево обычно ниже, чем red-black;
- поиск в среднем чуть приятнее;
- но цена — более внимательная поддержка структуры.

---

## 6. Почему нужны вращения

После вставки или удаления локальный баланс может нарушиться.

Нужно:

- не сломать порядок по ключам;
- уменьшить перекос;
- перестроить дерево только в малой окрестности проблемы.

Именно для этого служат вращения.

Вращение — это локальная перестройка нескольких вершин, которая:

- сохраняет in-order порядок;
- меняет форму дерева;
- улучшает баланс.

---

## 7. Самая важная мысль про вращения

Вращение не “сортирует дерево заново” и не меняет логического порядка ключей.

Оно делает только одно:

> меняет родительско-дочерние связи так, чтобы тяжёлая сторона не была слишком
> глубокой.

Это очень важно почувствовать именно структурно, а не как набор формул.

---

## 8. Малое правое вращение

Обычно нужно, когда вершина слишком тяжела влево, и её левый ребёнок тоже
тяготеет влево.

### Интуитивная картина

Было:

```text
      A
     /
    B
   /
  C
```

После правого вращения:

```text
      B
     / \
    C   A
```

Но на самом деле между этими вершинами есть поддеревья, и их тоже нужно
переставить аккуратно.

### Что происходит по смыслу

- левый ребёнок поднимается вверх;
- старая вершина уходит вправо;
- среднее поддерево переезжает так, чтобы сохранить BST-порядок.

---

## 9. Малое левое вращение

Это симметричный случай:

- вершина тяжела вправо;
- правый ребёнок тоже тяготеет вправо.

Было:

```text
  A
   \
    B
     \
      C
```

Стало:

```text
    B
   / \
  A   C
```

Опять же, важно помнить, что “среднее” поддерево не теряется, а аккуратно
переезжает на правильное место.

---

## 10. Почему вращение не ломает BST

Это один из самых фундаментальных фактов.

Рассмотрим правое вращение:

```text
        A
       /
      B
     / \
    T1  T2
```

и пусть справа у `A` есть `T3`.

Тогда до вращения порядок ключей:

```text
T1 < B < T2 < A < T3
```

После вращения:

```text
        B
       / \
      T1  A
         / \
        T2  T3
```

Порядок остаётся тем же:

```text
T1 < B < T2 < A < T3
```

То есть вращение меняет форму, но не in-order последовательность.

Вот почему вращения вообще допустимы в дереве поиска.

---

## 11. Двойные вращения

Иногда проблема имеет форму “зигзага”, и одного малого вращения недостаточно.

Это случаи:

- `LR` — левый ребёнок, но перекос внутри него вправо;
- `RL` — правый ребёнок, но перекос внутри него влево.

Тогда используют два вращения подряд.

### LR

1. малое левое вращение у левого ребёнка;
2. малое правое вращение у проблемной вершины.

### RL

1. малое правое вращение у правого ребёнка;
2. малое левое вращение у проблемной вершины.

---

## 12. Четыре классических случая

Очень полезно держать в голове таблицу:

| Перекос вершины | Перекос тяжёлого ребёнка | Что делать |
|---|---|---|
| LL | в ту же сторону | одно правое вращение |
| RR | в ту же сторону | одно левое вращение |
| LR | в противоположную | левое + правое |
| RL | в противоположную | правое + левое |

Это не просто “правила наизусть”, а отражение того, как именно нагрузка
распределилась в поддереве.

---

## 13. Как выбирать вращение интуитивно

Нужно задать два вопроса:

1. В какую сторону перекошена текущая вершина?
2. В какую сторону перекошен её тяжёлый ребёнок?

Если направления совпадают:

- достаточно одного малого вращения.

Если не совпадают:

- нужен двойной поворот, чтобы сначала выпрямить “зигзаг”, а потом уже
  выровнять общий перекос.

---

## 14. Реализация вращений на C++

```cpp
struct Node {
  int key;
  int height = 1;
  Node* left = nullptr;
  Node* right = nullptr;
};

int Height(Node* v) {
  return v == nullptr ? 0 : v->height;
}

void UpdateHeight(Node* v) {
  if (v != nullptr) {
    v->height = 1 + std::max(Height(v->left), Height(v->right));
  }
}

int BalanceFactor(Node* v) {
  return Height(v->right) - Height(v->left);
}

Node* RotateLeft(Node* x) {
  Node* y = x->right;
  Node* t2 = y->left;
  y->left = x;
  x->right = t2;
  UpdateHeight(x);
  UpdateHeight(y);
  return y;
}

Node* RotateRight(Node* y) {
  Node* x = y->left;
  Node* t2 = x->right;
  x->right = y;
  y->left = t2;
  UpdateHeight(y);
  UpdateHeight(x);
  return x;
}
```

---

## 15. Почему AVL кажется сложным в коде

Не потому, что сама идея очень глубока, а потому что в реализации нужно
аккуратно:

- обновлять высоты;
- возвращать новый корень поддерева;
- правильно переставлять поддеревья;
- различать четыре случая.

То есть сложность AVL — это в основном инженерная аккуратность.

---

## 16. Что важно запомнить

Про AVL-вращения нужно вынести три центральные идеи:

1. AVL строго контролирует разницу высот;
2. вращения — это локальный ремонт формы;
3. вращения не меняют порядок ключей, а только форму дерева.

Если эта тройка ясна, то вставка и удаление в AVL становятся намного понятнее,
потому что воспринимаются как:

> обычные BST-операции плюс локальное выпрямление дерева.
