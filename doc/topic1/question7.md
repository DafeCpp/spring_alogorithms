# Дек

## 1. Что такое дек

Дек (`deque`, double-ended queue) — это структура данных, у которой можно
эффективно работать с обоими концами.

Он поддерживает:

- добавление в начало;
- добавление в конец;
- удаление из начала;
- удаление из конца.

## 2. Как на него смотреть

Дек можно воспринимать как структуру, которая обобщает:

- очередь;
- стек.

Если алгоритму в разные моменты нужно работать то с одним, то с другим концом,
дек часто является естественным выбором.

## 3. Основные операции

| Операция | Смысл |
|---|---|
| `push_front(x)` | добавить в начало |
| `push_back(x)` | добавить в конец |
| `pop_front()` | удалить из начала |
| `pop_back()` | удалить из конца |
| `front()` | посмотреть начало |
| `back()` | посмотреть конец |

## 4. Чем дек отличается от очереди

Обычная очередь:

- добавляет в хвост;
- удаляет из головы.

Дек:

- умеет и добавлять, и удалять с обеих сторон.

Это делает его более гибким инструментом.

## 5. Реализация на циклическом массиве

Очень удобный способ:

- есть массив;
- есть два индекса;
- индексы двигаются по кругу по модулю размера.

Это даёт быстрые операции на концах.

## 6. Реализация на двусвязном списке

Тоже естественна:

- голова и хвост доступны напрямую;
- вставки и удаления на концах работают за `O(1)`.

## 7. Где дек особенно полезен

- монотонная очередь;
- задачи на скользящее окно;
- симуляции с двусторонней обработкой;
- алгоритмы, где нужно быстро выкидывать элементы с обоих концов.

## 8. Почему дек часто появляется в алгоритмах

Потому что многие онлайн-задачи требуют:

- держать актуальный диапазон элементов;
- быстро удалять устаревшие элементы с одного конца;
- быстро добавлять новые с другого.

Именно здесь дек оказывается незаменимым.

## 9. Сводная таблица практических характеристик

| Свойство | Оценка |
|---|---|
| Память | `O(n)` |
| Узнать длину | `O(1)`, если хранится `size`; иначе `O(n)` |
| Взять максимум | `O(n)` |
| Добавить элемент | `O(1)` в начало и `O(1)` в конец |

## 10. Что важно запомнить

Дек — это структура для двусторонней работы. Если стек слишком узок, а очередь
слишком жёстка, дек часто даёт именно ту гибкость, которая нужна.
